# Бот-перехватчик заказов для Telegram

Бот читает сообщения **только из указанных чатов** и отправляет подходящие сообщения в одну целевую группу.

## 1) Установка

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## 2) Настройка

1. Скопируйте `.env.example` в `.env`.
2. Заполните:
   - `BOT_TOKEN` — токен от `@BotFather`.
   - `SOURCE_CHAT_IDS` — ID чатов-источников через запятую.
   - `TARGET_CHAT_ID` — ID группы-приемника.
   - `ORDER_KEYWORDS` — ключевые слова для поиска заказов (или пусто, чтобы слать все).

## 3) Запуск

```powershell
python bot.py
```

## Ошибка `401 Unauthorized` / `InvalidToken`

Telegram не принял `BOT_TOKEN`. Чаще всего: токен отозван, скопирован с ошибкой или в `.env` лишние кавычки/пробелы.

1. Откройте `@BotFather` → ваш бот → **/token** (или **/revoke**, затем выдайте новый токен).
2. Вставьте токен в `.env` в одну строку, **без** кавычек: `BOT_TOKEN=123456:ABC...`
3. Если токен светился в логах или в чате — считайте его скомпрометированным и **обязательно перевыпустите**.

## Ошибка `ConnectError: All connection attempts failed`

Python **вообще не может открыть TCP** до `api.telegram.org` (не таймаут, а отказ соединения).

Частая причина: VPN работает **только для браузера**, а консоль и скрипты идут мимо VPN.

Что попробовать по порядку:

1. В VPN включите **TUN / системный режим / весь трафик** (не только браузер).
2. В `.env` укажите **локальный** HTTP-прокси клиента: `TELEGRAM_PROXY=http://127.0.0.1:7890` (порт возьмите из Clash / v2rayN / Hiddify).
3. Если в Windows уже задан **системный прокси**, добавьте `TELEGRAM_USE_SYSTEM_PROXY=1` — бот подхватит его из настроек ОС.
4. Либо задайте `HTTPS_PROXY=http://127.0.0.1:7890` в `.env`.

## Ошибка `TimedOut` / `ConnectTimeout`

Бот ходит на `api.telegram.org`. Если соединение режется или очень медленное, запрос не успевает за стандартное время.

1. Проверьте интернет и VPN (в некоторых сетях API Telegram недоступен без VPN).
2. В `.env` можно увеличить таймауты (секунды):  
   `TELEGRAM_CONNECT_TIMEOUT`, `TELEGRAM_READ_TIMEOUT`, `TELEGRAM_WRITE_TIMEOUT`, `TELEGRAM_POOL_TIMEOUT`.
3. Если нужен прокси: `TELEGRAM_PROXY=http://127.0.0.1:7890` (порт подставьте свой).  
   Для `socks5://` установите зависимость:  
   `pip install "python-telegram-bot[socks]"`.

## Важные условия работы

- Бот видит сообщения только в тех чатах, где он состоит.
- Для групп/супергрупп лучше дать боту права администратора.
- В `@BotFather` отключите Privacy Mode (`/setprivacy -> Disable`), иначе бот не увидит часть сообщений в группах.
- Для каналов бот должен быть добавлен в канал (обычно как администратор).

## Сообщение не пересылается

### Сообщения от **другого бота** (Mottor / Lpmotor и т.п.)

В Telegram **бот не получает** в API сообщения, которые отправили **другие боты** (в группах и супергруппах). Это ограничение платформы, **не лечится** настройкой `/setprivacy`.

Если заявки приходят от бота Mottor, ваш перехватчик **никогда** не увидит их через обычный Bot API.

**Что можно сделать:**

- Настроить в Lpmotor **уведомления на почту / webhook / другой канал**, не «второй бот в той же группе».
- Или использовать **клиент от имени пользователя** (Telethon / Pyrogram + `api_id` / `api_hash`) — это уже другой тип программы, не этот репозиторий.
- Или дублировать лиды туда, куда пишут **люди**, а не только сервисный бот.

### Остальные причины

1. **Ключевые слова** — совпадение **подстрока** в тексте (без учёта регистра). В тексте должна встретиться **целая фраза** из списка через запятую. Для заявок с сайта добавьте, например: `заявка`, `лид`, `lpmotor`.
2. **Privacy Mode** — в `@BotFather` → `/setprivacy` → **Disable**, иначе в **группах** бот часто **не видит** обычные сообщения **людей** (на сообщения от других ботов это не влияет).
3. **Чат** — заявка должна приходить в чат, чей ID указан в `SOURCE_CHAT_IDS`. Временно добавьте в `.env` строки `TELEGRAM_LOG_UNKNOWN_CHAT=1` и `TELEGRAM_DEBUG_FILTER=1`, перезапустите бота и посмотрите лог: будет видно `chat_id` и обрезок текста.
4. Бот должен быть **участником** чата-источника и целевой группы.

**Проверка «вручную»:** тестовое сообщение нужно отправить **в группу из `SOURCE_CHAT_IDS`**, а не в целевую (`TARGET_CHAT_ID`). В «Избранное» / личку боту текст из перехвата не попадает, если этих чатов нет в `SOURCE_CHAT_IDS`.

## Как узнать ID чатов

Простой способ:
1. Добавьте бота `@userinfobot` или любой аналог в нужный чат.
2. Получите `chat_id` (для групп обычно начинается с `-100...`).
3. Вставьте эти ID в `.env`.
